## What Was Wrong
### 1. Tracked generated bytecode
Tracked files under `__pycache__/` were not valid source and were removed.

### 2. Flat lifecycle mixing
The subtree did not distinguish:
- active evaluation entrypoints
- manual probes
- corpus/reference assets
- local-only/generated files

### 3. Corpus ownership ambiguity
Tracked corpus files under `eval/` visually compete with the packaged canonical dataset even though the README already says they are not the source of truth.

## Recommended Structure
Recommended grouped layout:
- `apps/xlightsdesigner-analysis-service/eval/README.md`
- `apps/xlightsdesigner-analysis-service/eval/manifest.v1.json`
- `apps/xlightsdesigner-analysis-service/eval/runners/`

- `apps/xlightsdesigner-analysis-service/eval/corpus/`
- `apps/xlightsdesigner-analysis-service/eval/archive/`

Purpose:
- `runners/`: promoted evaluation and benchmark entrypoints
- `probes/`: manual investigative tools
- `corpus/`: non-runtime reference corpus files
- `archive/`: superseded or zero-value probes after review
