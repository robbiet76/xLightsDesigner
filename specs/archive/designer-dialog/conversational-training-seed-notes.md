# Conversational Training Seed Notes

Status: Active
Date: 2026-03-13
Owner: xLightsDesigner Team

Purpose: capture the quality bar for the first curated `designer_dialog` conversational examples.

## Principles

- The user should be able to speak like a director, not an xLights operator.
- Prompts may be:
  - emotional
  - indirect
  - memory-driven
  - iterative
  - imperfect
- The designer should respond like a creative partner, not a scripted wizard.

## The Few-Shot Set Should Teach

1. When to proceed with bounded assumptions
2. When a single targeted clarification is necessary
3. How to use learned preferences softly
4. How to stay reviewable in degraded mode
5. When to avoid unnecessary designer detours and defer to direct technical sequencing
6. How to use scene/layout context to decide where broad coverage should live and where detail should focus
7. How to use music structure to decide where to hold, reveal, escalate, or simplify
8. How to keep sequence-local exceptions from silently rewriting broader preference memory

## Things To Avoid

- effect-name micromanagement in the designer examples
- xLights API wording
- style cloning from prior user sequences
- long questionnaires
- overconfident autonomous apply behavior

## Quality Bar

A strong designer response should:
- restate the design direction clearly
- make only the assumptions needed to move forward
- preserve novelty
- respect user taste without becoming repetitive
- remain concrete enough to produce a real brief and proposal
- use the spatial scene model intentionally rather than treating all props as interchangeable
- use the music arc intentionally rather than treating all sections as equal
