# xLightsDesigner Wireframes v5 (Creative Analysis + Chat-First Designer)

Status: Draft  
Date: 2026-03-05  
Purpose: Update wireframes to reflect Director/Designer model, Creative Analysis stage, and chat-first sequencing workflow.

## 1) Locked UX Decisions
- User is the director; agent is the designer.
- User input can be broad/subjective or specific.
- Chat is the primary sequencing interface.
- Design workspace does not expose full manual sequencing forms.
- Proposed changes are concise, human-readable summaries generated by the agent.
- No timeline/sequencer duplication from xLights.
- Creative Analysis is a first-class stage before major sequence generation.

## 2) Information Architecture
- Left navigation: `Project`, `Sequence`, `Design`, `History`, `Metadata`.
- Header (persistent): `Project`, `Active Sequence`, `xLights Status`, `Revision`, `Refresh`, `Review in xLights`.
- Global status bar under header: one-line status + `View Details`.
- Diagnostics: expandable drawer at bottom (collapsed by default).

## 3) Screen Specs

## 3.1 Project Screen
Sections:
- `Project Summary`: project name, show folder, xLights endpoint/version compatibility.
- `Project Settings`: discovery policy, retry policy, backup/revert policy.
- `Session State`: last active sequence, plan-only mode, last sync.

Primary controls:
- `Save Project Settings`
- `Refresh Connection`
- `Open in xLights`

## 3.2 Sequence Screen (Setup + Creative Analysis)
Sections:
- `Sequence Setup`:
  - open/select sequence,
  - media file confirmation,
  - sidecar metadata path summary.
- `Creative Analysis Kickoff`:
  - short guided prompt set for goals/style/inspiration,
  - freeform chat input for artistic direction.
- `Reference Media`:
  - upload image/video references,
  - list of attached references with remove/preview actions.
- `Creative Brief`:
  - user goals + inspiration summary,
  - section map candidates,
  - mood/energy arc,
  - lyric/theme narrative cues,
  - visual direction cues from uploaded references,
  - initial design hypotheses.

Primary controls:
- `Run Creative Analysis`
- `Regenerate Brief`
- `Accept Brief and Start Design`
- `Edit Brief Direction` (returns to kickoff prompts/chat)

Behavior:
- Creative Analysis can run with audio-only if lyrics/reference/internet context is unavailable.
- External research is optional and non-blocking.
- Reference uploads are stored under a Designer media folder within the active sequence folder.
- If reference media will be used directly in sequence output, file format must be validated against xLights-supported media formats before apply/export workflows.

## 3.3 Design Screen (Chat + Proposed Summary)
Layout:
- Left panel (persistent): `Designer Chat`
- Right panel: `Proposed Next Write` summary list

Left panel:
- conversation thread (user, designer agent, system),
- quick prompt chips,
- composer and send.

Right panel:
- single pick-list of proposed changes (agent-authored summaries),
- actions: `Add Line`, `Edit Selected`, `Remove Selected`,
- apply controls: `Apply to xLights`, `Discard Draft`.

Proposed row style:
- plain-language summary only (no required `Section/Model/Change` string format),
- optional compact context suffixes allowed when useful (labels/models impacted).

## 3.4 History Screen
Sections:
- version timeline (one version per approved agent update),
- selected version summary,
- rollback target selector.

Primary controls:
- `Rollback to Version`
- `Compare to Current`
- `Reapply as Variant`

## 3.5 Metadata Screen
Sections:
- tag library (curated + user-extensible),
- model/group semantic assignments,
- orphan handling and remap.

Primary controls:
- `Add Tag`
- `Assign Metadata`
- `Remap Orphan`

## 4) Primary Flow Contract
1. User configures project and opens sequence.
2. User provides kickoff direction (goals/inspiration), optionally uploads references.
3. Agent runs Creative Analysis and produces Creative Brief.
4. User accepts or refines brief.
5. User iterates in Design chat.
6. Agent maintains Proposed Next Write summaries.
7. User explicitly applies changes.
8. History records version checkpoint.

## 5) Control-State Contract

Common flags:
- `xlightsConnected`
- `activeSequenceLoaded`
- `creativeBriefReady`
- `hasDraftProposal`
- `proposalStale`
- `applyInProgress`
- `planOnlyMode`

Enablement:
1. `Run Creative Analysis`
- Enabled when sequence + media are loaded.

2. `Accept Brief and Start Design`
- Enabled when `creativeBriefReady = true`.

3. `Apply to xLights`
- Enabled when `hasDraftProposal = true` AND `xlightsConnected = true` AND `planOnlyMode = false` AND `proposalStale = false`.

4. `Rollback to Version`
- Enabled when selected version is not current head and apply not in progress.

## 6) Validation Contract
- Reference media validation:
  - store path must resolve under the active sequence folder media area,
  - media files flagged for sequence use must pass xLights-supported format checks,
  - unsupported formats can remain as inspiration-only references but are not eligible for sequence media use.
- Creative Analysis output must contain:
  - brief summary,
  - section candidates,
  - mood/energy cues,
  - at least one design-direction hypothesis.
- Proposal output must contain:
  - human-readable summary lines,
  - base revision token,
  - approx impact count.
- Apply is blocked on stale revision.
