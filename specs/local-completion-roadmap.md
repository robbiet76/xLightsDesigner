# Local Completion Roadmap

Status: Active
Owner: xLightsDesigner Team
Last Reviewed: 2026-04-30
Supersedes: dated local completion plans

## Purpose

Track the remaining local-first work needed before the app is a credible end-to-end product.

## Priorities

1. Project and show-folder lifecycle: reliable create/open/migrate flows, show-folder switching, and metadata refresh.
2. Display understanding: durable model metadata, custom model construction capture, submodel capture, and user-editable display semantics.
3. Sequence planning quality: full-display/full-song planning, better effect choices, stronger layer composition, and revision loops.
4. Review/apply safety: backups, scoped command review, validation, render proof, and restore.
5. History and recovery: compact event records, artifact references, and usable recovery workflows.
6. Training consolidation: keep promoted knowledge compact and remove generated-data clutter.
7. Release gates: repeatable app and xLights API validation on real show folders.

## Current Direction

Treat the local app as the reliable bridge between user intent, project metadata, trained knowledge, and xLights. Cloud and shared learning can build on this foundation later, but local project safety and repeatable validation come first.
